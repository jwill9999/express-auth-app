## [1.1.0] - 2026-02-21

### Added
- **JWT Lifecycle Hardening**: Full refresh token lifecycle with rotating tokens
  - `RefreshSession` domain entity with token family tracking
  - Rotating refresh tokens — each refresh issues a new token and revokes the old
  - Token reuse detection — if a rotated token is replayed, the entire family is revoked
  - `POST /auth/refresh` endpoint (accepts httpOnly cookie or request body)
  - `POST /auth/logout` endpoint (revokes current session)
  - `POST /auth/logout-all` endpoint (revokes all sessions for authenticated user)
  - `POST /auth/admin/revoke` endpoint (admin-forced revocation by userId)
  - Secure httpOnly cookie transport for refresh tokens
  - MongoDB TTL index for automatic expired session cleanup
  - 45 new tests covering all use cases, providers, and routes (86 total)

### Changed
- **Access Token TTL**: Default changed from 24h to 5m for short-lived access tokens
- **Login/Register responses**: Now include refresh token set as httpOnly cookie
- **AuthMiddleware**: Enforces strict `Bearer ` prefix parsing
- **Google OAuth callback**: Shares the same refresh session lifecycle as email/password login

### Security
- **Token rotation**: Every refresh rotates the token, preventing replay attacks
- **Reuse detection**: Replayed tokens trigger family-wide revocation
- **httpOnly cookies**: Refresh tokens stored in secure httpOnly cookies (not accessible to JS)
- **SHA-256 hashing**: Refresh tokens are hashed before storage in the database
- **Short-lived access tokens**: 5-minute default TTL reduces exposure window

## Upgrade Guide

### From 0.1.0 to 1.0.0

1. **Install Docker** (if not already installed)
2. **Create environment file**:
   ```bash
   cp .env.development.example .env.development
   ```
3. **Start with Docker**:
   ```bash
   docker-compose up -d
   ```

**Note**: Old JavaScript files can be removed after testing.
