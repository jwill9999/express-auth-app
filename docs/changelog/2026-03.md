## [1.1.3] - 2026-03-01

### Added

- **Rate Limiting Middleware** (`express-rate-limit`):
  - Auth endpoints (`/auth/*`): 5 requests per 15 minutes
  - Protected API endpoints (`/api/*`): 100 requests per 15 minutes
  - Returns `429 Too Many Requests` with `RateLimit-Limit`, `RateLimit-Remaining`, and `RateLimit-Reset` standard headers
  - `rateLimiting` flag on `AppDependencies` (defaults `true`; set to `false` in tests)
  - `RateLimitError` schema added to Swagger; `429` responses documented on `/auth/register` and `/auth/login`
  - Integration tests in `tests/interfaces/http/rateLimiter.test.ts`
- **GitHub Pages Documentation Site**:
  - Enabled GitHub Pages serving `docs/` from `main` branch
  - Added `docs/_config.yml` for Jekyll configuration (minima theme)
  - Live URL: [https://letuscode.co.uk/express-auth-app/](https://letuscode.co.uk/express-auth-app/)
  - All docs remain in repo — version-locked to code, PR-reviewed, AI-maintainable

## [1.2.0] - 2026-03-02

### Added

- **Unit test coverage ≥ 90%** across all Clean Architecture layers:
  - Domain: entities, value objects, error classes
  - Application: `RegisterUser`, `LoginUser` use cases with full branch coverage
  - Infrastructure: `BcryptPasswordHasher`, `JwtTokenProvider`, `MongoUserRepository`
  - Interfaces: controller integration tests, middleware, error mapping edge cases
  - Codecov badge added to README; `CODECOV_TOKEN` secret configured
- **CI/CD pipeline hardening** (`.github/workflows/`):
  - `ci.yml` — added `format:check` step, concurrency control, all actions pinned to full commit SHAs
  - `pr-checks.yml` (new) — Dependency Review blocks PRs introducing HIGH/CRITICAL CVE dependencies; fork-safe PR commenting
  - `docker.yml` (new) — Docker image build verification on every PR; Trivy container CVE scan on every PR + SARIF upload to GitHub Security tab on merge to main
  - `dependabot.yml` (new) — weekly automated PRs for npm and GitHub Actions SHA pins; grouped `@types/*` and `eslint*` updates
- **Branch protection rules** on `main`:
  - Required status checks: `Test & Coverage`, `Dependency Review`, `Docker Build`, `Analyze (javascript-typescript)`
  - 1 approving review required; stale reviews dismissed
  - Force pushes and branch deletion blocked; rules enforced for administrators
- **GitHub Security features** (configured in repository settings):
  - CodeQL Default Setup — SAST for JavaScript/TypeScript
  - Secret Scanning with push protection
  - Dependency Graph + Dependabot alerts
