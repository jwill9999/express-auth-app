## [1.1.3] - 2026-03-01

### Added

- **Rate Limiting Middleware** (`express-rate-limit`):
  - Auth endpoints (`/auth/*`): 5 requests per 15 minutes
  - Protected API endpoints (`/api/*`): 100 requests per 15 minutes
  - Returns `429 Too Many Requests` with `RateLimit-Limit`, `RateLimit-Remaining`, and `RateLimit-Reset` standard headers
  - `rateLimiting` flag on `AppDependencies` (defaults `true`; set to `false` in tests)
  - `RateLimitError` schema added to Swagger; `429` responses documented on `/auth/register` and `/auth/login`
  - Integration tests in `tests/interfaces/http/rateLimiter.test.ts`
- **GitHub Pages Documentation Site**:
  - Enabled GitHub Pages serving `docs/` from `main` branch
  - Added `docs/_config.yml` for Jekyll configuration (minima theme)
  - Live URL: [https://letuscode.co.uk/express-auth-app/](https://letuscode.co.uk/express-auth-app/)
  - All docs remain in repo — version-locked to code, PR-reviewed, AI-maintainable

## [1.3.0] - 2026-03-02

### Security

- **Removed committed `.env` files from git** — `.env.development` and `.env.production` were untracked. Added `.env.development.example` and `.env.production.example` as safe commit-safe templates. `.gitignore` updated to allow `!.env.*.example`.
- **`helmet` security headers** — `helmet` v8 installed and applied as the first Express middleware in `app.ts`. Sets `X-Content-Type-Options`, `X-Frame-Options`, `X-XSS-Protection`, `Referrer-Policy`, and more on every response.
- **Traefik reverse proxy (defense-in-depth headers)** — Traefik v3.3 replaces direct port exposure on the app container. Static config in `traefik/traefik.dev.yml` (HTTP + dashboard) and `traefik/traefik.prod.yml` (HTTPS + Let's Encrypt). Security header middleware (`frameDeny`, `contentTypeNosniff`, `browserXssFilter`, `referrerPolicy`, HSTS in prod) applied via Docker labels. App no longer exposes port 3000 directly.

### Added

- **Zod input validation at HTTP boundary** — `zod` v3 installed. `src/interfaces/http/validation/schemas.ts` defines schemas for `register`, `login`, and `adminRevoke` endpoints. `src/interfaces/http/validation/validate.ts` provides a reusable `validate(schema)` middleware factory. Invalid requests receive structured `400` responses with per-field error messages before reaching use cases.
- **Mongo integration tests** (`mongodb-memory-server`) — 14 new integration tests in `tests/integration/`:
  - `MongoUserRepository.integration.test.ts` (8 tests) — covers `save`, `findByEmail`, `findById`, `update`, unique email constraint, and email lowercasing.
  - `MongoRefreshSessionRepository.integration.test.ts` (6 tests) — covers `save`, `findByTokenHash`, `revokeByFamily`, `revokeAllByUserId`, and unique `tokenHash` constraint (index enforced via `syncIndexes()`).
- **`GET /health` endpoint** — Added to `src/interfaces/http/routes.ts`. Returns `{ status: "ok" }`. Used by Docker `HEALTHCHECK` and Traefik health probing.

### Changed

- **Dockerfile** — Added `USER node` (non-root) in runtime stage; added `HEALTHCHECK --interval=30s` using `wget`; added comment explaining SHA256 digest pinning.
- **`docker-compose.yml`** — Added `traefik` service (HTTP, dev dashboard on `:8080`); app service uses `expose` instead of `ports`; app healthcheck added.
- **`docker-compose.prod.yml`** — Added `traefik` service (HTTPS + Let's Encrypt, `letsencrypt_data` volume); app service uses `expose`; app healthcheck added. Requires `APP_DOMAIN` and `TRAEFIK_ACME_EMAIL` env vars.
